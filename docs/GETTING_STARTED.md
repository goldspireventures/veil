# Veil — getting started

Pick the path that matches your role. Each guide is short and step-by-step so you can self-serve without emailing support.

## I’m an IT admin setting up my company

1. Read **[ADMIN_GUIDE.md](ADMIN_GUIDE.md)** (15 min)
2. Open **[create team](https://veil.goldspireventures.com/create.html)** on the portal
3. Choose your **company type** — we apply the right default policy pack and enable related packs for departments
4. Save your **admin sign-in key** and **join code** (shown once)
5. Sign in at **[Admin](https://veil.goldspireventures.com/admin.html)** and follow the **Setup guide** on the Overview tab

**Typical timeline:** create org (5 min) → add member emails → share join code → members install → first security events in Admin within a day.

## I’m a team member

1. **[Install](https://veil.goldspireventures.com/install.html)** Veil from Chrome or Edge
2. Follow **[MEMBER_GUIDE.md](MEMBER_GUIDE.md)** — join code + work email from your admin (includes screenshots)
3. Refresh Outlook or Gmail after joining

## I’m using Veil just for myself (no company)

1. Install from the store → **[PERSONAL_GUIDE.md](PERSONAL_GUIDE.md)** (with screenshots)
2. No join code or admin portal needed

## Tech company with a Finance team?

- **Company default** (most employees): Engineering pack — set at create when you pick Technology / SaaS
- **Finance sub-team**: Admin → **Access** → enable **Finance** in pack library → **Settings** → create Finance sub-team → **Set policy** → assign finance members under **People**

See [ADMIN_GUIDE.md — Policy packs & sub-teams](ADMIN_GUIDE.md#4-policy-packs--sub-teams).
